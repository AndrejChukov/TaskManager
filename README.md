# Task Manager API 🚀

RESTful API приложение для управления задачами, написанное на **Spring Boot 3** и **Java 17**. Проект включает в себя систему авторизации на основе **JWT**, работу с базой данных **PostgreSQL**, механизм мягкого удаления (soft deleting) с автоматической очисткой, а также полную контейнеризацию с помощью **Docker**.

## 🛠 Технологический стек

*   **Язык:** Java 17
*   **Фреймворк:** Spring Boot 3.4.1 (Web, Data JPA, Security, Validation)
*   **База данных:** PostgreSQL 15, H2 (для тестов)
*   **Миграции:** Liquibase
*   **Безопасность:** Spring Security, OAuth2 Resource Server, JWT (Nimbus JOSE+JWT)
*   **Документация:** OpenAPI 3 (Swagger UI / springdoc-openapi)
*   **Тестирование:** JUnit 5, Mockito
*   **Инфраструктура:** Docker, Docker Compose, pgAdmin 4

## ✨ Ключевые возможности

*   **Аутентификация и авторизация:**
    *   Регистрация и логин с выдачей JWT токена.
    *   Разделение ролей: `ADMIN` и `USER`.
    *   Кастомная валидация (например, запрет использования слова "admin" в имени пользователя).
*   **Управление задачами:**
    *   Пользователи могут создавать, просматривать, редактировать и удалять свои задачи.
    *   Задачи имеют статусы (`NEW`, `IN_PROGRESS`, `DONE`) и приоритеты (`LOW`, `MEDIUM`, `HIGH`).
    *   Пагинация и сортировка списков задач.
*   **Soft Delete и фоновые задачи:**
    *   При удалении задача помечается как удаленная (`deleted = true`), а не стирается из БД сразу.
    *   Реализован `TrashPurgeService`, который автоматически безвозвратно удаляет задачи, находившиеся в корзине более 30 дней.
*   **Управление пользователями (Админ-панель):**
    *   Администратор имеет доступ к просмотру всех пользователей, назначению задач любым пользователям и удалению аккаунтов.
*   **Docker среда:**
    *   Проект поднимается одной командой вместе с БД и преднастроенным **pgAdmin** (сервер БД уже добавлен в интерфейс с помощью `servers.json`).

## Быстрый старт

### Требования
*   Установленный [Java 17 JDK](https://adoptium.net/)
*   Установленный [Maven](https://maven.apache.org/) (или используйте wrapper `./mvnw`)
*   [Docker](https://docs.docker.com/get-docker/) и [Docker Compose](https://docs.docker.com/compose/install/)

### 🚀 Инструкция по запуску

1. **Клонируйте репозиторий:**
   ```bash
   git clone https://github.com/AndrejChukov/TaskManager
   cd taskmanager
   ```

2. **Соберите JAR-файл приложения:**
   Поскольку `Dockerfile` копирует готовый артефакт из папки `target`, проект нужно предварительно скомпилировать.
   ```bash
   mvn clean package -DskipTests
   ```

3. **Запустите контейнеры:**
   ```bash
   docker-compose up --build -d
   ```

4. **Проверьте статус приложения:**
   Перейдите по адресу: [http://localhost:8080/status](http://localhost:8080/status)
   Вы должны увидеть ответ: `System is online`.

## 📦 Инфраструктура и сервисы

После успешного запуска `docker-compose`, вам будут доступны следующие сервисы:

| Сервис | Адрес | Описание |
| :--- | :--- | :--- |
| **API Backend** | `http://localhost:8080` | Основное Spring Boot приложение |
| **Swagger UI** | `http://localhost:8080/swagger-ui/index.html` | Интерактивная документация API |
| **PostgreSQL** | `localhost:5432` | База данных (`user: myuser`, `pass: mypassword`) |
| **pgAdmin 4** | `http://localhost:5050` | Веб-интерфейс для работы с БД |

### Доступ к pgAdmin
*   **Логин:** `admin@admin.com`
*   **Пароль:** `admin`
*   *Сервер `TaskManager` уже преднастроен и готов к работе.*

## 📚 Документация API (Endpoints)

Подробная документация доступна в Swagger UI после запуска проекта. Ниже представлен краткий обзор:

### Auth (Открытые эндпоинты)
*   `POST /api/auth/register` — Регистрация нового пользователя.
*   `POST /api/auth/signin` — Вход и получение JWT токена.

### Задачи пользователя (Требуется токен авторизации)
*   `GET /api/my-tasks` — Получить список своих задач.
*   `POST /api/my-tasks` — Создать задачу для текущего пользователя.
*   `PUT /api/my-tasks/{id}` — Обновить свою задачу.
*   `DELETE /api/my-tasks/{id}` — Удалить свою задачу.

### Администрирование (Требуется роль `ROLE_ADMIN`)
*   `GET /api/users` — Получить список всех пользователей (с пагинацией).
*   `GET /api/users/{id}` — Получить пользователя по ID.
*   `POST /api/users` — Создать пользователя напрямую.
*   `DELETE /api/users/{id}` — Удалить пользователя.
*   `GET /api/tasks` — Получить список всех задач в системе.
*   `GET /api/tasks/user/{id}` — Получить задачи конкретного пользователя.
*   `POST /api/tasks/{id}` — Создать задачу для конкретного пользователя.

## 🧪 Тестирование

```bash
mvn test
```
Тесты покрывают бизнес-логику в сервисах (`TaskServiceTest`, `UserServiceTest`), используя **Mockito**.
